# Пользователи и роли

## Ролевая модель

В системе предусмотрены две роли:

- **Администратор** – полный доступ ко всем разделам, включая управление пользователями.
- **Менеджер по персоналу** – доступ к кадровым операциям, но не к управлению пользователями.

## Создание первого администратора

При первом запуске в базе данных существует Главный Администратор. Для входа следует использовать:

- Логин: admin,
- Пароль: admin.

## Управление пользователями через интерфейс

1. Администратор заходит в раздел **«Пользователи»**.

2. Видит таблицу:

- ID,
- ФИО,
- логин,
- роль.

3. Может добавить нового пользователя

- Нажать кнопку **Добавить пользователя**,
- Заполнить форму (ФИО, ввести логин и пароль, выбрать роль),
- Нажать кнопку **Сохранить**(пароль хешируется автоматически).

4. Может редактировать пользователя (изменить любое поле).

5. Может удалить пользователя (мягкое удаление – доступ теряется, запись остаётся в БД).
